# Task 05: Расширение day0_checks.py под триггеры и прогрев

**Story:** story-voice-trigger-warmup.md
**Роль-исполнитель:** Claude Code (сценарий), Андрей (запуск и подтверждение)
**Приоритет:** средний — закрывает story, фиксирует «не переливитировать» факты
**Зависимости:** task-01, task-02, task-03, task-04 (проверяет результат всех
предыдущих задач)

## Описание

Добавить в `day0_checks.py` проверки, специфичные для hardware/окружения
Андрея, которые нельзя достоверно выполнить в обычной разработческой среде:

1. Замер `load_duration` при вызове `warm_up_model()` на реальной RTX 5070 Ti
   — фиксирует эмпирическое время прогрева, используется для калибровки
   таймаута `WARMING → Error` из task-02.
2. Проверка, что глобальный хоткей (task-03) действительно перехватывается
   вне фокуса приложения — включая проверку конфликта с другим ПО, если
   регистрация не удалась.
3. Проверка, что клик по орбу (task-04) и хоткей приводят к идентичному
   поведению (один и тот же путь состояния).
4. Проверка отсутствия дублирующих запросов прогрева при быстром повторном
   триггере (двойной клик / двойное нажатие хоткея подряд).

## Реализация (набросок)

Следует существующему паттерну `day0_checks.py` — rerunnable, печатает
понятный PASS/FAIL по каждому пункту, не требует ручной интерпретации логов.
Пункт 1 (замер `load_duration`) — не pass/fail, а печать конкретного числа,
которое Андрей затем вручную вносит в конфиг таймаута.

## Критерии приёмки

- [ ] Скрипт перезапускаем без побочных эффектов (как остальные day0-проверки).
- [ ] Результат замера `load_duration` явно выведен и снабжён рекомендацией
      («установите таймаут WARMING не меньше X с запасом»).
- [ ] Конфликт регистрации хоткея (если есть другое ПО с той же комбинацией)
      даёт понятное сообщение, а не тихий провал.

## Stop condition / hardware handoff

Это по сути и есть hardware-handoff задача целиком — скрипт пишет Claude
Code, но **выполняет и подтверждает результаты Андрей** на своей машине с
реальным микрофоном, реальной клавиатурой и реальной RTX 5070 Ti. После
подтверждения — обновить `PROJECT.md` замеренными числами как «verified,
do not re-litigate» факт.
