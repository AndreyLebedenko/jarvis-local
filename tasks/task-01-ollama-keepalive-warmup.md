# Task 01: Настраиваемый keep_alive + асинхронный вызов прогрева Ollama

**Story:** story-voice-trigger-warmup.md
**Роль-исполнитель:** Claude Code
**Приоритет:** высокий (независимая, самая дешёвая часть story)
**Зависимости:** нет — можно делать первой, не дожидаясь UI-задач

## Описание

Две независимые, но связанные вещи:

1. Вынести `keep_alive` в текущий `config.py` (`BackendSettings`), вместо
   неявного дефолта Ollama (5 минут). Значение передаётся в каждый запрос к
   `/api/chat`.
2. Добавить функцию `warm_up_model()`, которая шлёт запрос с пустым
   `messages: []` и текущим `keep_alive`, ничего не генерируя — только
   загружает модель в VRAM. Вызывается асинхронно (fire-and-forget), не
   блокируя основной поток UI/VAD.

## Реализация (набросок)

Использовать существующий стек проекта: `httpx`, `OllamaBackend`,
`BackendSettings`, `logging`, тесты через `httpx.MockTransport`. Не вводить
`aiohttp` и не создавать второй Ollama client рядом с `backend.py`.

`warm_up_model()` должен быть тонким методом/функцией вокруг существующего
backend adapter. Ошибка прогрева не должна ронять UI/VAD-поток, но ее нельзя
глотать молча: она логируется и уходит в будущие «События системы» как
WARN/ERROR уровня warmup, после чего обычный voice pipeline продолжает жить.

## Критерии приёмки

- [ ] `keep_alive` читается из конфига, значение по умолчанию — предмет
      отдельной эмпирической проверки Андреем (открытый вопрос из story card).
- [ ] `warm_up_model()` не блокирует вызывающий поток и использует текущий
      `OllamaBackend`/`httpx` stack.
- [ ] Повторный вызов `warm_up_model()` во время уже идущей загрузки модели не
      создаёт гонки/дублирующих запросов (простой guard-флаг достаточен).
- [ ] Сбой запроса (Ollama не запущена, таймаут) не приводит к падению
      приложения и не скрывается молча — пишется понятная запись в лог событий.

## Stop condition / hardware handoff

Нет hardware-зависимой части — можно проверить полностью в обычной
разработческой среде без участия Андрея. Handoff не требуется, но стоит,
чтобы Андрей замерил реальное время прогрева (`load_duration` из ответа
Ollama) на своей RTX 5070 Ti — это значение понадобится в task-02 для расчёта,
насколько заранее нужно триггерить прогрев относительно завершения фразы.
