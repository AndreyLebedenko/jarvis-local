# Task 04: Клик по статус-орбу как ручной триггер

**Story:** story-voice-trigger-warmup.md
**Роль-исполнитель:** Claude Code
**Приоритет:** низкий-средний (дешёвая задача, хороший universal fallback)
**Зависимости:** task-01, task-02

## Описание

Добавить обработчик клика на существующий компонент статус-орба, вызывающий
тот же путь триггера, что и Push-to-Talk (`trigger_warmup()` →
`WARMING` → захват речи). В отличие от глобального хоткея, это работает
через обычный `onClick` уже сфокусированного окна — не требует OS-level
глобальных хуков и поэтому не зависит от проблем X11/Wayland, описанных в
task-03. Хороший запасной путь для будущего Linux-порта, пока хоткей там не
отлажен.

## Реализация (набросок)

Тривиально с точки зрения кода — orb уже существует как UI-компонент;
добавляется единственный обработчик, который вызывает ту же функцию, что и
`WindowsHotkeyProvider.on_trigger` в task-03. Стоит явно вынести
`on_activation_trigger()` как общую точку входа, чтобы все три источника
(хоткей, клик, будущий wake word) сходились в одном месте, а не дублировали
логику по отдельности.

## Критерии приёмки

- [ ] Клик по орбу в состоянии `IDLE` переводит его в `WARMING` тем же путём,
      что и хоткей.
- [ ] Повторный клик во время `WARMING`/`LISTENING` не создаёт дублирующих
      запросов прогрева (тот же guard, что в task-01).
- [ ] Логика триггера вынесена в общую функцию, используемую всеми
      источниками активации.

## Stop condition / hardware handoff

Не требуется — чисто UI-логика, полностью проверяется в обычной среде
разработки без участия Андрея.
